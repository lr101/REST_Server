**********************************
Database (sensors):

Table -> id:
--------------------
1. sensorID --- primary key
2. sensorNick
3. sensorType

Table -> {sensorID}
--------------------
1. date
2. value
3. rowID --- primary key


***********************************
REST-API:
host adress: {host-IP}:{host-port}/sensors/

Table -> id:
---------------------
GET /id/     					//returns the complete Table -> id 
GET /id/{sensorID}				//returns row with of the given sensorID
GET /id/{sensorID}/sensorType			//returns sensorType of specific sensorID
GET /id/{sensorID}/sensorNick			//returns sensorType of specific sensorID
POST /id/					//add new Sensor to Table -> id and create Table -> {sensorID} body: {"sensorID":"{sensorID}", "sensorNick":"{sensorNick}", "sensorType":"{sensorType}"}
DELETE /id/{sensorID}/				//delete Table -> {sensorID} and entry in Table -> id *restricted access

Table -> {sensorID}
----------------------
GET /{sensorID}/				//returns the complete Table -> {sensorID}
GET /{sensorID}?start={date1}&end={date2}	//returns the table from date1 until date2
POST /{sensorID}/				//adds a new table entry with body as JSON: {"date":"{date1}", "value":"{value1}"} 
DELETE /{sensorID}?start={date1}&end={date2}	//delete entries from date1 to date2. If no dates are given, all entries are deleted
